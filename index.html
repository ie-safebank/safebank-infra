The 5 Implemented OpenSSF Frameworks

The OpenSSF framework ‘Concise Guide for Developing More Secure Software’ is a list of 27 guidelines aimed at reducing vulnerabilities attackers could exploit. This section describes the security guidelines implemented in Safe-Bank.

  1. Multi Factor Authentication

    MFA adds an additional security layer to Github accounts, reducing risks of unauthorized access. The Cybersecurity engineer activated the “Requirement to have MFA” to ensure organizational security.

  2. Monitor Dependency Vulnerabilities

    IE-Safebank uses Dependabot in Github to scan for vulnerabilities categorized as low, medium, high, or critical. Medium and low risks were deprioritized due to time constraints.

  3. Keep Dependencies Reasonably Up-to-Date

    To address human-centric aspects of Dependabot, alerts were connected to a Slack channel to notify developers promptly.

  4. Do Not Push Secrets to a Repository

    Using Github’s ‘Secret Scanning’ tool, commits are automatically scanned for hard-coded secrets, reducing vulnerabilities.

  5. Review Code Changes

    IE-Safebank enforces code reviews before merging pull requests. This practice enhances main branch security and fosters knowledge sharing among team members.

Learning Secure Software Development Practices

IE-Safebank prioritizes secure development. The Cybersecurity engineer completed the Linux Foundation’s hands-on course with top scores to ensure robust practices.
